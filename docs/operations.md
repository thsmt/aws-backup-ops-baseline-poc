# 運用

## 概要

バックアップは作成して終わりではなく、対象管理、通知確認、復旧確認、保持期間の見直しが重要です。

このドキュメントでは、AWS Backup を運用する際に確認するポイントを整理します。

<br>

## バックアップ対象の管理

バックアップ対象はタグで管理します。

| キー | 値 |
| --- | --- |
| `Backup` | `true` |

リソース追加時は、対象リソースにタグが付いているか確認します。

バックアップ対象外にしたいリソースには、このタグを付けないか、別のタグ値にします。

<br>

## 定期確認項目

| 項目 | 確認内容 |
| --- | --- |
| Backup Job | 失敗、期限切れ、中断がないか |
| Recovery Point | 想定どおり作成されているか |
| Backup Vault | 復旧ポイント数と保持期間が適切か |
| 通知 | SNS購読やChatOps通知が有効か |
| タグ | 新規リソースにバックアップ対象タグが付与されているか |
| 復旧手順 | 復旧テストの実施記録があるか |

<br>

## 保持期間の見直し

保持期間は、復旧要件とコストの両方に影響します。

| 観点 | 確認内容 |
| --- | --- |
| RPO | どの時点まで戻せる必要があるか |
| RTO | どのくらいの時間で復旧する必要があるか |
| 監査要件 | 何日、何か月、何年保持する必要があるか |
| コスト | 復旧ポイント増加による保管費用 |

<br>

## Vault Lock 利用時の注意

Vault Lock の `compliance` は強力ですが、猶予期間後は解除できません。

本番で利用する前に、検証環境で保持期間、削除手順、復旧手順を確認します。

<br>

## 削除時の注意

CloudFormation スタックを削除しても、Backup Vault 内に復旧ポイントが残っている場合は Backup Vault を削除できません。

このテンプレートでは、Backup Vault を `Retain` として扱い、意図しない復旧ポイント削除を避けます。
